# STEP 4 — DOMAIN MODEL DEFINITIONS

Define the following domain primitives.

---

## ParticipantId

Represents the unique identifier for a participant.

Requirements:

• strongly typed identifier  
• immutable  
• GUID based

Example fields:

Id

---

## ParticipantRole

Represents the role of the participant inside the ecosystem.

Roles may include:

Participant  
Worker  
Operator  
Investor  
Administrator  

Must be implemented as a **value object or enum**.

---

## ParticipantStatus

Represents the lifecycle state of a participant.

Statuses:

PendingRegistration  
Active  
Suspended  
Disabled  

---

## ParticipantProfile

Participant identity metadata.

Fields:

ParticipantId  
FirstName  
LastName  
Email  
PhoneNumber  
Country  
CreatedAt  

Profile must be immutable once created except through aggregate operations.
