# 2.1.28 WORKFORCE LIFECYCLE ENGINE

You are implementing the **Workforce Lifecycle Engine**.

This engine manages **state transitions of workforce members** within the HEOS system.

Lifecycle control governs workforce participation across the Whycespace economy.

Examples of lifecycle states:

• Registered  
• Active  
• Unavailable  
• Suspended  
• Terminated  

The engine ensures transitions follow valid operational rules.

The engine operates on:

WorkforceAggregate

Domain models exist in:

src/domain/core/workforce/

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

---

# STEP 7 — LIFECYCLE TRANSITION RULES

The engine must enforce valid transitions.

Allowed transitions:

Registered → Active  
Active → Unavailable  
Active → Suspended  
Unavailable → Active  
Suspended → Active  
Active → Terminated  

Forbidden transitions:

Terminated → Active  
Suspended → Registered  
Unavailable → Registered  

Termination is final.

Reactivation is only allowed from Suspended.

Lifecycle actions may require operator authorization.

Reactivate → requires manager-level authority  

Authorization validation should occur before lifecycle mutation.

WorkforceLifecycleEngine → another engine

Runtime orchestration must invoke engines.
