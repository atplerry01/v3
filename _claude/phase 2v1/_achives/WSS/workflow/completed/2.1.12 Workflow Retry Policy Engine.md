# WHYCESPACE WBSM v3
# PHASE 2.1.12 — WORKFLOW RETRY POLICY ENGINE

You are implementing the Workflow Retry Policy Engine for the Whyce Structural System (WSS).

This engine manages retry behavior for workflow steps that fail during execution.

It ensures failed workflow steps can be retried according to defined failure policies without breaking workflow execution.

# SYSTEM CONTEXT

System: Whyce Structural System (WSS)  
Tier: T1M — Orchestration  

Workflow steps may fail during execution.

Failure policies define how the system reacts.

Possible behaviors:

Retry  
Skip  
Fail  
Compensate  

The Retry Policy Engine evaluates retry decisions.

---

# OBJECTIVES

Implement an engine that can:

1 Evaluate retry policies for failed workflow steps
2 Track retry attempts
3 Determine whether a step should retry
4 Determine retry delay
5 Stop retries after max attempts
6 Produce retry decisions

# ENGINE IMPLEMENTATION

Create:

WorkflowRetryPolicyEngine.cs

Responsibilities:

EvaluateRetryPolicy

• determine if retry should occur  
• compare retry count with MaxRetries  
• return RetryDecision  

RegisterRetryAttempt

• increment retry counter in retry store  

GetRetryCount

• retrieve retry count from retry store  

ResetRetryCount

• reset retry counter after successful step completion  

The engine must remain stateless.

Retry counters are stored externally.
