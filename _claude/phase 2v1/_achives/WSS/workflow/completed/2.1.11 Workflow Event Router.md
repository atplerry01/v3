# WHYCESPACE WBSM v3
# PHASE 2.1.11 — WORKFLOW EVENT ROUTER

You are implementing the Workflow Event Router for the Whyce Structural System (WSS).

The Workflow Event Router is responsible for routing workflow events between:

• workflow runtime
• execution engines
• the global event fabric (Kafka)

This component ensures workflow execution remains fully event-driven.

# SYSTEM CONTEXT

System: Whyce Structural System (WSS)  
Tier: T1M — Orchestration  

WSS operates as an event-driven runtime.

Workflow execution emits events that must be routed to:

• execution engines
• workflow state updates
• external event fabric

The router integrates with the **Global Event Fabric (Kafka)**.

---

# OBJECTIVES

Implement a router that can:

1 Route workflow execution events
2 Publish workflow events to Kafka
3 Deliver events to workflow runtime subscribers
4 Deliver events to engine execution layer
5 Support event subscription
6 Provide deterministic routing

# ROUTER IMPLEMENTATION

Create:

WorkflowEventRouter.cs

Responsibilities:

PublishEvent

• construct WorkflowEventEnvelope
• publish event through Kafka publisher
• attach workflow metadata

Subscribe

• allow runtime components to subscribe to events
• maintain in-memory handler registry

RouteInternalEvent

• deliver event to registered handlers
• trigger next workflow step via runtime dispatcher

The router must NOT contain workflow business logic.

Routing only.
