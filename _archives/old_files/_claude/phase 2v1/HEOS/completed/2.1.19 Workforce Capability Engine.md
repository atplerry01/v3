# WHYCESPACE WBSM v3
# HEOS — HUMAN ECONOMIC OPERATING SYSTEM
# 2.1.19 WORKFORCE CAPABILITY ENGINE

You are implementing the **Workforce Capability Engine**.

This engine manages **capabilities (skills)** assigned to workforce members within HEOS.

Capabilities determine which **economic tasks** a worker can perform inside the Whycespace system.

Examples:

• Driving  
• PropertyMaintenance  
• CustomerSupport  
• Logistics  
• Operations  

The engine operates on the **WorkforceAggregate** defined in:

src/domain/core/workforce/

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

---

# STEP 1 — ARCHITECTURE VERIFICATION

Component classification:

Workforce Capability Engine = **Execution Engine**

Correct layer:

src/engines/T2E/

Reason:

Capability assignment is deterministic business logic applied to the WorkforceAggregate.

It does not orchestrate workflows.

It does not manage distributed execution.

Therefore it belongs in **T2E Execution Engines**.

---

# STEP 7 — CAPABILITY RULES

The engine must enforce:

Duplicate capabilities are not allowed.

Capabilities may only be assigned if worker status is **Active**.

Suspended workers cannot receive new capabilities.

Removing the final capability is allowed only if worker remains registered.

Capabilities must exist in the **domain capability list**.

---

# STEP 8 — ENGINE INVOCATION RULE

The engine must never call other engines.

Correct invocation flow:

Runtime Dispatcher → WorkforceCapabilityEngine

Incorrect:

WorkforceCapabilityEngine → another engine

Runtime orchestration is responsible for engine invocation.
