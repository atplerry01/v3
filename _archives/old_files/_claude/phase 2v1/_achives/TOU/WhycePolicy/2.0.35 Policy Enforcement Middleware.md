# WHYCESPACE WBSM v3
# PHASE 2.0.35 — POLICY ENFORCEMENT MIDDLEWARE

You are implementing Phase 2.0.35 of the Whycespace system.

This phase introduces the **Policy Enforcement Middleware**.

This middleware enforces WHYCEPOLICY decisions before system operations execute.

All protected operations must pass through this middleware.

---

# ARCHITECTURE RULES

ENGINE → stateless logic  
SYSTEM → stateful storage

The middleware must be stateless.

# ENGINE

Create:

PolicyEnforcementEngine.cs

Responsibilities:

Perform policy evaluation and determine whether an operation is allowed.

Steps:

1 Build PolicyContext

2 Retrieve cached decision if available

3 If cache miss:
   run PolicyEvaluationEngine

4 Return enforcement result

---

# ENFORCEMENT RULES

If any policy decision = deny

Result:

Allowed = false

If all policies allow or log

Result:

Allowed = true

---

# MIDDLEWARE

Create:

PolicyEnforcementMiddleware.cs

Purpose:

Intercept platform requests before execution.

Steps:

1 Extract ActorId and Domain

2 Build PolicyEnforcementRequest

3 Call PolicyEnforcementEngine

4 If denied:
      return HTTP 403

5 If allowed:
      pass request to next middleware
