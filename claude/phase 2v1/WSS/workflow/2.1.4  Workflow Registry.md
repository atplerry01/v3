# WHYCESPACE WBSM v3
# PHASE 2.1.4 — WORKFLOW REGISTRY

You are implementing the Workflow Registry for the Whyce Structural System (WSS).

The registry stores and manages executable WorkflowDefinitions used by the runtime.

Workflows may originate from:

• direct definitions
• generated templates

The registry becomes the central lookup service for the workflow runtime.

---

# ARCHITECTURE RULES

Follow WBSM v3.

• Engines are stateless  
• Engines never call engines  
• Registry stores definitions only  
• Workflow execution belongs to runtime  
• Validation must occur before registration
