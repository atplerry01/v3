# WHYCESPACE WBSM v3
# PHASE 2.1.13 — WORKFLOW TIMEOUT ENGINE

You are implementing the Workflow Timeout Engine for the Whyce Structural System (WSS).

This engine monitors workflow execution and detects when workflow steps or entire workflow instances exceed allowed execution time.

Timeout handling prevents workflows from hanging indefinitely and ensures system stability.

# SYSTEM CONTEXT

System: Whyce Structural System (WSS)  
Tier: T1M — Orchestration  

Workflow steps may have execution time limits.

If a step or workflow exceeds the configured timeout, the engine must detect this and trigger timeout handling.

Timeout detection supports distributed workflow execution.

---

# OBJECTIVES

Implement an engine that can:

1 Register workflow step timeouts
2 Register workflow instance timeouts
3 Monitor execution duration
4 Detect timeout violations
5 Produce timeout decisions
6 Integrate with workflow event routing

# ENGINE IMPLEMENTATION

Create:

WorkflowTimeoutEngine.cs

Responsibilities:

RegisterStepTimeout

• store step timeout configuration in store

RegisterWorkflowTimeout

• store workflow-level timeout

CheckStepTimeout

• retrieve TimeoutEntry
• determine if timeout exceeded
• return TimeoutDecision

CheckWorkflowTimeout

• check workflow-level timeout
• return TimeoutDecision

ClearTimeout

• remove timeout tracking after step completion

The engine must remain stateless.

Timeout data must be stored in the timeout store.
