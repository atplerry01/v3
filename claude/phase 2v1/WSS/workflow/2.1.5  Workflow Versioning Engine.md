# WHYCESPACE WBSM v3
# PHASE 2.1.5 — WORKFLOW VERSIONING ENGINE

You are implementing the Workflow Versioning Engine for the Whyce Structural System (WSS).

This engine manages versioned workflow definitions and ensures safe upgrades of workflows without breaking existing running workflow instances.

---

# ARCHITECTURE RULES

Follow WBSM v3 strictly.

• Engines are stateless  
• Engines never call engines  
• Versioning applies only to workflow definitions  
• Running workflow instances must remain bound to their original version  
• Version resolution must be deterministic

# STORAGE

Create:

WorkflowVersionStore.cs

Use:

ConcurrentDictionary<string, SortedDictionary<string, WorkflowDefinition>>

Structure:

workflowId
    version → WorkflowDefinition

SortedDictionary ensures latest version lookup.

---

# VERSION VALIDATION

Version must:

• follow semantic format
• not duplicate existing version
• belong to the correct workflowId

Reject invalid version formats.

---

# VERSION RESOLUTION

When retrieving latest workflow:

Use highest semantic version.

Example:

Registered:

1.0.0  
1.1.0  
1.2.0  

Latest = 1.2.0
