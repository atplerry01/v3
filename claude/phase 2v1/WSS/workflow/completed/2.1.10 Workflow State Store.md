# WHYCESPACE WBSM v3
# PHASE 2.1.10 — WORKFLOW STATE STORE

You are implementing the Workflow State Store for the Whyce Structural System (WSS).

This component persists workflow execution state during runtime.

The store tracks workflow instance progress, completed steps, and execution metadata.

It allows the workflow runtime to resume execution and query workflow progress.

---

# WBSM v3 ARCHITECTURE RULES

Follow WBSM v3 strictly.

• Domain models live in the system layer  
• Engines and stores live in the engines layer  
• Stores manage state  
• Engines remain stateless  
• State lookup must be deterministic  
• State operations must be thread-safe  

Workflow state persistence supports distributed workflow execution.

# STORE IMPLEMENTATION

Create:

WorkflowStateStore.cs

Use:

ConcurrentDictionary<string, WorkflowState>

Structure:

instanceId → WorkflowState

Responsibilities:

SaveState

• persist initial workflow state

GetState

• retrieve workflow execution state

UpdateState

• update step progress
• update workflow status
• update UpdatedAt timestamp

AddCompletedStep

• record completed step
• prevent duplicate step entries

DeleteState

• remove workflow state after completion

ListActiveStates

• return workflows where status = Running
