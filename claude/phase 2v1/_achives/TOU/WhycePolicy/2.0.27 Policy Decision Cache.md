# WHYCESPACE WBSM v3
# PHASE 2.0.27 — POLICY DECISION CACHE

You are implementing Phase 2.0.27 of the Whycespace system.

This phase introduces the **Policy Decision Cache**, which stores recent policy evaluation results to improve performance.

Policy evaluation may occur thousands or millions of times per second.

Caching avoids repeated evaluation for identical contexts.

---

# ARCHITECTURE RULES

ENGINE → stateless logic  
SYSTEM → stateful storage

The cache itself is a **system store**.

# CACHE KEY RULES

Cache key must include:

domain
actorId
attribute hash

Example key:

identity:123:9F4A82C1

Attribute hash must be deterministic.

---

# TTL

Default cache TTL:

60 seconds

After TTL expires, cache entry must be invalidated.
