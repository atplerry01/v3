# WHYCESPACE WBSM v3
# PHASE 2.0.32 — POLICY ROLLOUT ENGINE

You are implementing Phase 2.0.32 of the Whycespace system.

This phase introduces the **Policy Rollout Engine**.

The rollout engine controls how policies are gradually deployed across the system.

Policies must not always activate globally.

Rollouts allow progressive deployment.

---

# STRATEGY RULES

Global

Policy applies everywhere.

Percentage

Policy applies to a deterministic percentage of actors.

Use deterministic hash of ActorId.

Example:

hash(actorId) % 100 < percentage

ActorList

Policy applies only to listed actors.

DomainList

Policy applies only to listed domains.
