# 2.1.27 WORKFORCE COMPLIANCE ENGINE

You are implementing the **Workforce Compliance Engine**.

This engine evaluates whether workforce members comply with operational rules, governance requirements, and workforce policies within the HEOS system.

Compliance evaluation ensures:

• workforce operational integrity  
• policy adherence  
• assignment eligibility  
• incentive eligibility  
• lifecycle governance  

This engine performs **compliance evaluation only**.

It must **NOT mutate domain aggregates**.

The engine operates on:

WorkforceAggregate

Domain models exist in:

src/domain/core/workforce/

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

# STEP 7 — COMPLIANCE RULES

The engine must enforce:

Worker must have **Active status** for normal operation.

Suspended or terminated workers must fail compliance.

Capability mismatches must be flagged.

Excessive failed tasks must reduce compliance score.

Workers operating outside authorized scope must trigger violation.

Compliance period must be valid.

---

# STEP 8 — ENGINE INVOCATION RULE

The engine must never call other engines.

Correct invocation flow:

Runtime Dispatcher → WorkforceComplianceEngine

Incorrect:

WorkforceComplianceEngine → another engine

Runtime orchestration must invoke engines.
