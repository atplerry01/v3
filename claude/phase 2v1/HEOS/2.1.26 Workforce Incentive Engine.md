# WHYCESPACE WBSM v3
# HEOS — HUMAN ECONOMIC OPERATING SYSTEM
# 2.1.26 WORKFORCE INCENTIVE ENGINE

You are implementing the **Workforce Incentive Engine**.

This engine converts workforce performance outcomes into **economic incentive decisions** inside the HEOS system.

Incentives may later be routed into the Whycespace economic system, including:

• workforce rewards  
• bonus allocations  
• performance payouts  
• contribution incentives  

This engine evaluates workforce incentive eligibility and computes deterministic incentive outcomes.

Performance inputs are provided externally and may come from the Workforce Performance Engine output, but this engine must NOT call other engines directly.

The engine operates on:

WorkforceAggregate

Domain models exist in:

src/domain/core/workforce/

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

# STEP 1 — ARCHITECTURE VERIFICATION

Component classification:

Workforce Incentive Engine = **Execution Engine**

Correct layer:

src/engines/T2E/

Reason:

This engine performs deterministic business execution that converts performance metrics into an incentive decision.

It is not a pure analytics engine because it produces an executable economic decision.

It does not orchestrate workflows.

Therefore it belongs in **T2E Execution Engines**.

---

# STEP 7 — INCENTIVE RULES

The engine must enforce:

Worker must have **Active workforce status**.

Evaluation period must be valid.

PerformanceScore must be within allowed range.

BaseIncentiveAmount must be greater than or equal to zero.

Suspended or terminated workers are not eligible for incentives.

Incentive amount calculation must be deterministic.

Example calculation concept:

BaseIncentiveAmount × performance multiplier

Example multipliers:

Low → 0.0  
Standard → 1.0  
High → 1.25  
Exceptional → 1.5  

Reject unknown performance tiers.

---

# STEP 8 — ECONOMIC OUTPUT RULE

This engine produces an **economic decision**, not a direct payout.

It must NOT transfer money.

It must only return a deterministic incentive decision that can later be consumed by:

• runtime workflows  
• vault transaction workflows  
• revenue distribution pipelines  

This preserves WBSM separation of concerns.

---

# STEP 9 — ENGINE INVOCATION RULE

The engine must never call other engines.

Correct invocation flow:

Runtime Dispatcher → WorkforceIncentiveEngine

Incorrect:

WorkforceIncentiveEngine → another engine

Runtime orchestration must invoke engines.
