# WHYCESPACE WBSM v3
# HEOS — HUMAN ECONOMIC OPERATING SYSTEM
# 2.1.23 ASSIGNMENT ENGINE

You are implementing the **Assignment Engine**.

The Assignment Engine allocates workforce members to operational tasks inside the Whycespace ecosystem.

Assignments may originate from:

• SPV operational tasks  
• cluster activities  
• workflow jobs  
• service dispatch systems  

The engine determines whether a worker is eligible and assigns the task.

The engine operates on:

WorkforceAggregate  
OperatorAggregate

Domain models exist in:

src/domain/core/workforce/  
src/domain/core/operators/

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

---

# STEP 1 — ARCHITECTURE VERIFICATION

Component classification:

Assignment Engine = **Execution Engine**

Correct layer:

src/engines/T2E/

Reason:

Assignment logic is deterministic business execution.

It allocates workers to tasks but does not orchestrate workflows.

Therefore it belongs in **T2E Execution Engines**.

# STEP 6 — ENGINE IMPLEMENTATION

Create:

AssignmentEngine.cs

Responsibilities:

• verify workforce eligibility  
• verify workforce availability  
• verify capability match  
• verify operator authorization  
• assign worker to task

Example method:

AssignWorker(
    WorkforceAggregate workforce,
    OperatorAggregate operator,
    AssignmentCommand command)

Return:

AssignmentDecision

Engine requirements:

stateless  
thread-safe  
deterministic  

The engine must NOT:

persist state  
call other engines  
invoke infrastructure

---

# STEP 7 — ASSIGNMENT RULES

The engine must enforce:

Worker must have **Active status**.

Worker must be **Available**.

Worker must possess required **capability**.

Operator must have authority for the assignment scope.

Worker cannot be assigned if already Busy.

TaskType must correspond to worker capability.

---

# STEP 8 — ENGINE INVOCATION RULE

The engine must never call other engines.

Correct invocation flow:

Runtime Dispatcher → AssignmentEngine

Incorrect:

AssignmentEngine → another engine

Runtime orchestration must invoke engines.
