# WHYCESPACE WBSM v3
# HEOS — HUMAN ECONOMIC OPERATING SYSTEM
# 2.1.18 WORKFORCE REGISTRY

You are implementing the **Workforce Registry** inside the Whycespace system.

The Workforce Registry is part of **HEOS (Human Economic Operating System)**.

It represents the **domain structure for participants who become workforce members** within the Whycespace economic system.

Workforce members may later be assigned to:

• SPVs  
• Clusters  
• Operational roles  
• Economic tasks  
• Scheduling systems  

The Workforce Registry extends the **Participant system**.

Participants must first exist inside the **Participant Registry** before joining the workforce.

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

# STEP 6 — AGGREGATE OPERATIONS

Implement domain operations.

RegisterWorker

Creates workforce record for an existing participant.

AssignCapability

Adds a capability to the workforce member.

RemoveCapability

Removes a capability.

ActivateWorker

Moves status to Active.

SuspendWorker

Suspends workforce participation.

TerminateWorker

Ends workforce membership.

All operations must enforce domain invariants.

---

# STEP 7 — DOMAIN INVARIANTS

Enforce the following rules:

Participant must exist before workforce registration.

Duplicate capabilities are not allowed.

Worker must be Active before assignments.

Terminated workers cannot be reactivated.

Suspended workers cannot receive new capabilities.
