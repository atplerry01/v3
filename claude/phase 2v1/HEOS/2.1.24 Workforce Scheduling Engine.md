# WHYCESPACE WBSM v3
# HEOS — HUMAN ECONOMIC OPERATING SYSTEM
# 2.1.24 WORKFORCE SCHEDULING ENGINE

You are implementing the **Workforce Scheduling Engine**.

This engine manages **time-based scheduling of workforce assignments** within the HEOS system.

Scheduling coordinates workforce availability across time windows for operational tasks.

Examples of scheduling use cases:

• worker shift scheduling  
• planned maintenance operations  
• scheduled property inspections  
• workforce rotations  
• long-duration task assignments  

The engine operates on:

WorkforceAggregate  
OperatorAggregate

Domain models exist in:

src/domain/core/workforce/  
src/domain/core/operators/

This implementation must follow **WBSM v3 Prompt Structural Rules and Guardrails**.

# STEP 1 — ARCHITECTURE VERIFICATION

Component classification:

Workforce Scheduling Engine = **Execution Engine**

Correct layer:

src/engines/T2E/

Reason:

Scheduling logic is deterministic execution logic applied to workforce and operator aggregates.

It does not orchestrate workflows and does not perform distributed runtime orchestration.

Therefore it belongs in **T2E Execution Engines**.

---

# STEP 7 — SCHEDULING RULES

The engine must enforce:

Worker must have **Active workforce status**.

Worker must be **Available** for the scheduled time window.

ScheduleEnd must be later than ScheduleStart.

Worker cannot have overlapping schedules.

Operator must have authority over the schedule scope.

Scheduled tasks must match workforce capability.

---

# STEP 8 — ENGINE INVOCATION RULE

The engine must never call other engines.

Correct invocation flow:

Runtime Dispatcher → WorkforceSchedulingEngine

Incorrect:

WorkforceSchedulingEngine → another engine

Runtime orchestration must invoke engines.
