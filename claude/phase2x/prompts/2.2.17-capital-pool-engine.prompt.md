# STEP 3 — PURPOSE OF CAPITAL POOL ENGINE

Capital pools represent **aggregated capital funds** used for deployment into economic activities.

Examples:

- Investor capital pools
- SPV investment pools
- CWG pooled capital
- Strategic investment pools

The engine performs operations such as:

- create capital pool
- activate pool
- suspend pool
- close pool

The engine must produce deterministic domain events for every mutation.

---

# STEP 4 — ENGINE INPUT COMMANDS

Commands are received via EngineContext.Data dictionary with an "action" field to route:

Create: poolId, poolName, clusterId, subClusterId, spvId, currency, createdBy
Activate: poolId, currentPoolStatus, activatedBy
Suspend: poolId, currentPoolStatus, reason, suspendedBy
Close: poolId, currentPoolStatus, closedBy
