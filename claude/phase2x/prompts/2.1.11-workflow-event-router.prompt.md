## Component Purpose

The Workflow Event Router connects the event fabric (Kafka) to workflow execution progression. When an event is emitted by an execution engine, the event router determines whether the event affects a workflow instance. If the event matches a workflow step dependency, the router advances the workflow to the next step.

## Architecture Constraints

- Runtime orchestration only — no business logic
- No direct engine invocation
- Uses system-layer interfaces for instance registry and state store
- Thread-safe (ConcurrentDictionary for idempotency)
- Correlation-based routing via WorkflowCorrelationId
- Idempotent event processing (duplicate events are ignored)
