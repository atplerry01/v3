# IDENTITY ACCESS SCOPE ENGINE

This prompt must be executed using the Whycespace MASTER PROMPT GENERATION TEMPLATE.

This component implements the Identity Access Scope Engine responsible for assigning and revoking access scopes for identities.

Access scopes define the boundaries within which permissions are valid.

Examples:

cluster:mobility
cluster:property
spv:property-acquisition
domain:governance
system:whycepolicy

claude/bootstrap-prompts/master-bootstrap.prompt.md

must be respected.

SECTION 2 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

Identity Access Scope Engine performs controlled mutation of identity access scopes.

Responsibilities:

• assign access scopes to identities
• revoke access scopes
• validate scope structure
• emit scope change events

The engine must NOT:

• persist data
• access infrastructure directly
• call other engines

Persistence occurs through system stores.
