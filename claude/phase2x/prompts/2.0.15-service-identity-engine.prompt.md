# SERVICE IDENTITY ENGINE

This prompt must be executed using the Whycespace MASTER PROMPT GENERATION TEMPLATE.

This component implements the Service Identity Engine responsible for managing identities assigned to services and system components operating inside the Whycespace ecosystem.

Service identities represent non-human system actors such as microservices, infrastructure modules, automation agents, and integration connectors.

Service identities are used to authenticate and authorize system components communicating across the platform.

SECTION 2 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

Service Identity Engine performs operational identity lifecycle management for system services.

Responsibilities:

• register service identities
• generate service credentials
• validate service identity status
• revoke service identity access
• emit service identity events

Service Identity Engine must NOT:

• modify human identity records
• access infrastructure directly
• perform persistence logic
• call external services

The engine produces service identity lifecycle decisions and events only.
