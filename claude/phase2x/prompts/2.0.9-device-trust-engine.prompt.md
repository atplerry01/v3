# DEVICE TRUST ENGINE

This prompt must be executed using the Whycespace MASTER PROMPT GENERATION TEMPLATE.

This component implements the Device Trust Engine responsible for evaluating the trustworthiness of devices used by identities.

Device trust evaluation determines whether a device should be considered trusted when performing authentication or authorization operations.

Trust evaluation considers:

device fingerprint
device history
geolocation stability
device reputation
usage patterns

claude/bootstrap-prompts/master-bootstrap.prompt.md

must be respected.

------------------------------------------------------------

• evaluate device trust score

• mutate identity state
• persist data
• call other engines
• access infrastructure directly

SECTION 6 — TRUST SCORE CALCULATION
------------------------------------------------------------

Device trust must be calculated using weighted scoring.

Example factors:

Known device: +20

Stable fingerprint: +20

Consistent geolocation: +10

Device usage history: +20

Low-risk IP reputation: +15

Device age: +15

Maximum score: 100

Trust levels:

0–39 = Low

40–69 = Medium

70–100 = High
