src/engines/T0U/Governance/GovernanceProposalTypeEngine.cs

## Purpose

Manages governance proposal categories — validates, registers, and deactivates proposal types within the governance system. Ensures proposals comply with constitutional governance rules.

## Commands

| Command | Description |
|---|---|
| `RegisterProposalTypeCommand` | Register a new governance proposal type |
| `DeactivateProposalTypeCommand` | Deactivate an existing proposal type |
| `ValidateProposalTypeCommand` | Validate a proposal type is registered and active for a domain |

## Events Emitted

| Event | Trigger |
|---|---|
| `GovernanceProposalTypeRegisteredEvent` | Successful registration |
| `GovernanceProposalTypeDeactivatedEvent` | Successful deactivation |
| `GovernanceProposalTypeValidatedEvent` | Successful validation |

## Execution Pattern

Returns `(GovernanceProposalTypeResult, Event?)` tuples. Null event on failure, populated event on success.

## Validation Rules

- Proposal type string must not be empty
- Description / reason must not be empty
- Guardian must exist and be Active
- Registration: type must not already exist
- Deactivation: type must exist and be active
- Validation: type must be registered and active; authority domain required

## Dependencies

- `GovernanceProposalTypeStore` — in-memory read model for proposal types
- `GuardianRegistryStore` — guardian existence and status verification
