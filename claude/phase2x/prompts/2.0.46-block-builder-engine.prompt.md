------------------------------------------------------------
SECTION 1 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

You are implementing the **Block Builder Engine** for the WhyceChain system.

WhyceChain provides immutable evidence infrastructure for the Whycespace ecosystem.

The Block Builder Engine constructs **Chain Blocks** from ledger entries.

A block groups ordered ledger entries and produces:

- deterministic block hash
- Merkle root for entries
- entry count
- block height
- linkage to previous block

This engine prepares blocks for:

- integrity verification
- Merkle proof generation
- external chain anchoring

The engine does NOT perform persistence or anchoring.

SECTION 6 — BLOCK CONSTRUCTION RULES
------------------------------------------------------------

The block builder must perform:

1. entry validation
2. deterministic ordering
3. Merkle root generation

Block fields to populate:

BlockId, BlockHeight, PreviousBlockHash, Entries, MerkleRoot, BlockHash, EntryCount, CreatedAt

------------------------------------------------------------
SECTION 7 — MERKLE ROOT GENERATION
------------------------------------------------------------

Merkle tree rules:

- leaf nodes = entry hashes
- internal nodes = SHA256(left + right)
- if node count is odd -> duplicate last node

Merkle root must be deterministic across nodes.

------------------------------------------------------------
SECTION 8 — BLOCK HASH RULE
------------------------------------------------------------

BlockHash must be computed using SHA256 over:

BlockHeight, PreviousBlockHash, MerkleRoot, EntryCount, CreatedAt

Hashing must use canonical serialization.

------------------------------------------------------------
SECTION 9 — ENGINE EXECUTION
------------------------------------------------------------

Engine: BlockBuilderEngine.cs

Primary method: Execute(BlockBuilderCommand command)

Execution steps:

1. validate ledger entries
2. extract entry hashes
3. compute Merkle root
4. compute block hash
5. construct ChainBlock
6. return BlockBuilderResult

The engine must not mutate ledger entries.
