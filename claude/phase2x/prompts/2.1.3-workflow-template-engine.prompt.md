COMPONENT PURPOSE
------------------------------------------------------------

The Workflow Template Engine creates reusable workflow blueprints used by the WSS orchestration system.

Workflow templates define standardised workflow structures that can be instantiated repeatedly with different parameters.

The engine produces immutable template definitions. It does NOT execute workflows.

------------------------------------------------------------
ENGINE RESPONSIBILITIES
------------------------------------------------------------

The engine must:

- create workflow templates
- validate template structure
- validate template steps
- validate dependency configuration (circular dependency detection via Kahn's algorithm)
- validate parameter placeholders
- generate deterministic TemplateId (SHA256 of name + version + structure)
- construct immutable WorkflowTemplateResult objects

The engine must NOT:

- persist templates
- execute workflows
- call runtime components
- invoke other engines

------------------------------------------------------------
ARCHITECTURE CONSTRAINTS
------------------------------------------------------------

Engine is:

- sealed, stateless, deterministic, thread-safe
- implements IEngine (ExecuteAsync)
- decorated with [EngineManifest] attribute
- T1M orchestration tier, Decision kind
