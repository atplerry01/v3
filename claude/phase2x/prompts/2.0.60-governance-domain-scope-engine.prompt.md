# 2.0.60 — Governance Domain Scope Engine

## Component

- **Number**: 2.0.60
- **Name**: GovernanceDomainScopeEngine
- **System**: Guardian Governance System
- **Layer**: ENGINE (T0U — Constitutional)
- **Category**: Execution Engine

## Location

```
src/engines/T0U/Governance/GovernanceDomainScopeEngine.cs
```

## Purpose

Manages governance domain authorization — validates that governance proposals remain within correct domain scope, registers new governance domains, and deactivates existing domains.

## Commands

| Command | Description |
|---------|-------------|
| `RegisterDomainScopeCommand` | Register a new governance domain scope |
| `DeactivateDomainScopeCommand` | Deactivate an existing governance domain |
| `ValidateDomainScopeCommand` | Validate that a proposal is within an allowed domain scope |

## Events

| Event | Trigger |
|-------|---------|
| `GovernanceDomainScopeRegisteredEvent` | Domain scope registered |
| `GovernanceDomainScopeDeactivatedEvent` | Domain scope deactivated |
| `GovernanceDomainScopeValidatedEvent` | Domain scope validated for proposal |

## Result

`GovernanceDomainScopeResult` — includes Success, AuthorityDomain, ProposalId, ProposalType, Action, Message, ExecutedAt.

## Dependencies

- `GovernanceDomainScopeStore` (system layer)
- `GuardianRegistryStore` (system layer)

## Validation Rules

- Authority domain must exist
- Domain must not be deactivated
- Requesting guardian must exist and be active
