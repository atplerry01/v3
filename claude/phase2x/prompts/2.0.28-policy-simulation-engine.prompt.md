# 2.0.28 — Policy Simulation Engine

## Component
- **Name:** Policy Simulation Engine
- **System:** WhycePolicy System
- **Layer:** Engines Layer
- **Tier:** T0U (Constitutional Governance Engine)
- **Category:** Intelligence Engine

## Purpose
Evaluates policy outcomes in simulation mode without affecting the production governance pipeline. Enables governance operators to test new policies, simulate changes, forecast outcomes, validate dependencies, and detect unintended effects.

## Files
- `src/engines/T0U/WhycePolicy/PolicySimulationInput.cs`
- `src/engines/T0U/WhycePolicy/PolicySimulationResult.cs`
- `src/engines/T0U/WhycePolicy/PolicySimulationRecord.cs`
- `src/engines/T0U/WhycePolicy/PolicySimulationEngine.cs`
- `tests/whycepolicy/PolicySimulationEngineTests.cs`

## Debug Endpoint
- `POST /dev/policy/simulate/batch` — Accepts `PolicySimulationInput`, returns `PolicySimulationResult`

## Rules
- Stateless, deterministic, thread-safe
- No persistence, no infrastructure access
- Simulation results are read-only
- No domain events emitted
- Never mutates policy state
