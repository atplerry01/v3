COMPONENT PURPOSE
------------------------------------------------------------

The Workflow Timeout Engine detects and evaluates timeout conditions for workflow steps and workflow instances.

In distributed systems, workflow steps may stall due to:

• network failures
• service outages
• unresponsive execution nodes
• external system delays

Timeout detection ensures workflows do not remain indefinitely blocked.

The engine determines whether a step or workflow has exceeded its allowed execution time.

------------------------------------------------------------
ENGINE RESPONSIBILITIES
------------------------------------------------------------

The engine must:

• evaluate step timeout policies
• determine whether timeout thresholds are exceeded
• detect stalled workflow steps
• calculate timeout durations
• produce timeout evaluation results

The engine must NOT:

• execute workflow steps
• persist workflow state
• interact with runtime dispatchers
• call other engines

Timeout state persistence is handled by the **Workflow State Store**.

------------------------------------------------------------
ENGINE EXECUTION LOGIC
------------------------------------------------------------

Execution Flow:

1. Load timeout policy
2. Calculate elapsed execution time
3. Compare elapsed time with timeout duration
4. Determine timeout status
5. Produce timeout evaluation result

Timeout evaluation rules:

• timeout occurs when elapsed time > TimeoutDuration
• no timeout occurs if execution is within allowed duration
• timeout detection must be deterministic

ElapsedTime calculation:

ElapsedTime = CurrentTimestamp - StepStartedAt
