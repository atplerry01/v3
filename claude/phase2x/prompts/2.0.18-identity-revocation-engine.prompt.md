## System Context

The **Identity Revocation Engine** is responsible for evaluating and
authorizing revocation of identities or identity capabilities.

Revocation scenarios include:
- compromised identity
- fraud detection
- policy enforcement
- governance suspension
- regulatory restriction
- role/privilege revocation
- device compromise

Revocation does NOT mutate domain aggregates directly.
The engine produces deterministic revocation decisions which
the runtime layer later converts into events and state changes.

## Engine Input

### IdentityRevocationRequest

Fields:
- IdentityId
- RequestedAt
- RevocationReason
- RequestedBy
- RevocationScope
- Evidence
- SourceSystem

### RevocationScope enum
- FullIdentityRevocation
- RoleRevocation
- PermissionRevocation
- DeviceRevocation
- SessionRevocation
- CredentialRevocation

### RevocationReason enum
- CompromisedAccount
- FraudDetected
- PolicyViolation
- RegulatorySuspension
- ManualGovernanceAction

## Engine Execution

1. Validate identity existence
2. Validate current identity status
3. Evaluate revocation scope
4. Evaluate revocation reason
5. Evaluate evidence validity
6. Evaluate governance override conditions
7. Determine revocation approval

## Engine Output

### IdentityRevocationResult

Fields:
- RevocationId (deterministic SHA256)
- IdentityId
- Approved
- RevocationScope
- RevocationReason
- RiskScore
- RequiresGovernanceReview
- Reason
- EvaluatedAt

RevocationId = SHA256(identityId + requestedAt + revocationScope)

## Event Output

IdentityRevocationEvaluatedEvent (emitted by runtime, not engine):
- EventId, IdentityId, RevocationId, Approved, RevocationScope,
  RevocationReason, RiskScore, Timestamp
