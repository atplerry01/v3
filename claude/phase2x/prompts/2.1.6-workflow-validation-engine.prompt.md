COMPONENT PURPOSE
------------------------------------------------------------

The Workflow Validation Engine verifies that workflow definitions, graphs,
and templates meet structural and orchestration requirements before being
accepted into the WSS system.

The validation engine acts as a gatekeeper ensuring only valid workflows
enter the Workflow Registry and runtime orchestration environment.

Validation protects the system from:

- invalid step definitions
- missing engine mappings
- circular dependencies
- invalid retry policies
- invalid timeout configurations
- parameter mismatch errors

------------------------------------------------------------
ENGINE RESPONSIBILITIES
------------------------------------------------------------

The engine must:

- validate workflow definitions
- validate workflow graph integrity (DAG)
- validate template structures
- validate step dependency references
- validate engine mapping references
- validate retry and timeout configurations
- validate parameter definitions
- produce validation reports

The engine must NOT:

- persist workflow data
- execute workflows
- invoke runtime components
- call other engines
- access infrastructure

- RetryPolicy (MaxRetries, RetryDelaySeconds, CompensationStepId)
- Timeout (TimeoutSeconds)

Fields:

- Warnings

------------------------------------------------------------
VALIDATION RULES
------------------------------------------------------------

- All dependency references must exist
- Graph must be acyclic (DAG)
- Engine names must not be empty
- MaxRetries: 0-10
- RetryDelaySeconds: 0-1800
- Timeout: 1s-86400s (24h)
- Parameter names must be unique and non-empty
- Parameter types must be valid (string, int, decimal, bool, datetime, guid)
- CompensationStepId must reference a valid step

access infrastructure, call other engines
