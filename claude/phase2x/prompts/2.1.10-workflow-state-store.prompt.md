------------------------------------------------------------
COMPONENT PURPOSE
------------------------------------------------------------

The Workflow State Store maintains the runtime execution state of workflow instances.
It tracks step-level execution progress, allowing the WSS runtime to persist and recover workflow execution state.

------------------------------------------------------------
STORE OPERATIONS
------------------------------------------------------------

- PersistWorkflowState: Insert or update workflow execution state
- UpdateStepState: Update step execution state by instance ID and step ID
- GetWorkflowState: Retrieve workflow state by instance ID
- ListPendingSteps: List pending steps for an instance
- ListCompletedSteps: List completed steps for an instance

------------------------------------------------------------
ARCHITECTURE CONSTRAINTS
------------------------------------------------------------

- Contains persistence logic only
- Deterministic, no business logic
- No engine invocations
- No runtime orchestration logic
- No workflow execution
