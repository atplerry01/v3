# 2.0.30 — Policy Impact Forecast Engine

## Component

- **Name:** Policy Impact Forecast Engine
- **System:** WhycePolicy System
- **Layer:** Engines Layer
- **Tier:** T3I (Governance Intelligence Engine)
- **Category:** Intelligence Engine

## Purpose

Predicts governance and operational impact of policy changes before deployment. Evaluates how proposed policy changes affect policy decisions, workflow outcomes, cluster governance rules, SPV execution rules, vault operations, and cross-cluster policy interactions.

## Path

`src/engines/T3I/WhycePolicy/`

## Files

| File | Purpose |
|------|---------|
| `PolicyImpactForecastInput.cs` | Input: CurrentPolicies, ProposedPolicies, SimulationContexts |
| `PolicyImpactForecastResult.cs` | Result: ForecastRecords, AffectedPolicies, RiskLevel, GeneratedAt |
| `PolicyImpactRecord.cs` | Impact record with ImpactType and Severity enums |
| `PolicyImpactForecastEngine.cs` | Engine: ForecastImpact method |

## Algorithm

1. Evaluate current policies for each simulation context
2. Evaluate proposed policies for same context
3. Compare decisions
4. Identify decision changes
5. Classify impact types (NO_CHANGE, DECISION_CHANGE, ESCALATION_CHANGE, ACCESS_CHANGE, GOVERNANCE_CHANGE)
6. Assign severity levels (LOW, MEDIUM, HIGH, CRITICAL)
7. Generate forecast records

## Rules

- Stateless, deterministic, thread-safe
- No persistence, no infrastructure access
- Does not mutate policy definitions
- Does not execute governance workflows
- Does not emit domain events

## Debug Endpoint

`POST /dev/policy/forecast/impact` — accepts `PolicyImpactForecastInput`, returns `PolicyImpactForecastResult`
