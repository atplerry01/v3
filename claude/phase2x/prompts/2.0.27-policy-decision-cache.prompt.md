SECTION 2 — COMPONENT PURPOSE
------------------------------------------------------------

The Policy Decision Cache stores previously computed policy decisions.

The goal of the cache is to improve performance by avoiding repeated policy evaluation when the same context is evaluated multiple times.

The cache is optional but improves runtime efficiency when:

• repeated commands trigger identical policy contexts
• high-frequency operations require rapid policy checks
• governance decisions must be returned quickly

The cache must never replace the canonical policy evaluation.

If a cache entry is missing or expired, the Policy Evaluation Engine must be executed.

------------------------------------------------------------

• TTL configurable
• stale entries automatically ignored
• cache must never alter decision logic
