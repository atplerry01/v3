SECTION 2 — COMPONENT PURPOSE
------------------------------------------------------------

The Policy Context Engine constructs the **policy evaluation context** used by the WhycePolicy system.

Policies require contextual information to evaluate conditions.

Examples of context attributes:

• identity attributes (from WhyceID)
• resource metadata
• action type
• cluster domain
• SPV identifiers
• vault identifiers
• governance metadata
• runtime attributes

The Policy Context Engine collects and structures these attributes into a deterministic evaluation context.

This engine does NOT evaluate policies.

It prepares context for the Policy Evaluation Engine.

SECTION 6 — ENGINE IMPLEMENTATION
------------------------------------------------------------

Create file:

PolicyContextEngine.cs

Primary method:

BuildContext(PolicyContextInput input)

Responsibilities:

• validate context fields
• normalize attribute names
• enforce deterministic attribute ordering
• generate immutable PolicyContext object

Rules:

• missing attributes allowed
• attributes must be normalized to lowercase keys
• context must be immutable

------------------------------------------------------------
SECTION 7 — ENGINE RULES
------------------------------------------------------------

This engine must follow WBSM v3 execution rules.

The engine must be:

• stateless
• deterministic
• thread-safe

The engine must NOT:

• persist data
• call other engines
• access infrastructure
• mutate domain aggregates

The engine only prepares evaluation context.
