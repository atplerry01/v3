## Location

```
src/engines/T0U/Governance/GovernanceProposalEngine.cs
```

## Responsibilities

Manages governance proposal lifecycle initiation:

- Create governance proposals with validation
- Submit proposals for governance review
- Cancel governance proposals
- Emit proposal lifecycle events

## Commands

- `CreateGovernanceProposalCommand` — Creates a new proposal (title, description, type, authority domain, proposer)
- `SubmitGovernanceProposalCommand` — Submits a draft proposal for review
- `CancelGovernanceProposalCommand` — Cancels a proposal with reason

## Events

- `GovernanceProposalCreatedEvent`
- `GovernanceProposalSubmittedEvent`
- `GovernanceProposalCancelledEvent`

## Result

- `GovernanceProposalResult` — Success, ProposalId, ProposalType, AuthorityDomain, Action, Message, ExecutedAt

## Validation Rules

### Creation
- Title must not be empty
- Description must not be empty
- ProposalType must be valid enum
- AuthorityDomain must be defined
- ProposedByGuardianId must be valid (non-empty GUID)
- ProposalId must be valid and unique within command context

### Submission
- Proposal must exist in registry
- Proposal must be in Draft status
- SubmittedByGuardianId must be valid

### Cancellation
- Proposal must exist in registry
- Proposal must not be finalized (Approved or Closed)
- Proposal must not already be cancelled
- CancelledByGuardianId must be valid
- Reason must be provided

## Architecture Compliance

- Engine is `sealed`, stateless, deterministic
- No persistence logic in Execute methods
- No engine-to-engine calls
- Events are immutable records
- Communication via events only
- Follows command -> result -> event pattern
