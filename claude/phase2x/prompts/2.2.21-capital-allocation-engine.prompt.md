## Purpose

Capital allocation converts reserved capital into allocated capital assigned to a specific execution target (SPV, Asset, Project, OperationalProgram). Allocation represents the formal assignment of capital to an economic activity. Once allocated, capital becomes available for utilization by execution workflows.

## Runtime Execution Flow

Command → WSS → Runtime Dispatcher → CapitalAllocationEngine → Domain Event → Event Fabric (Kafka) → Projection Workers → Capital Registry Update

## Governance Integration

Runtime pipeline enforces WhyceID (identity verification) and WhycePolicy (policy evaluation) before engine execution. Engine does NOT implement governance policies.
