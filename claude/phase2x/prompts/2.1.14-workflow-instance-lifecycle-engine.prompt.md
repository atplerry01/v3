------------------------------------------------------------
COMPONENT PURPOSE
------------------------------------------------------------

The Workflow Instance Lifecycle Engine governs the complete lifecycle of workflow instances.

It determines transitions between lifecycle states such as:

- workflow creation
- workflow start
- workflow completion
- workflow failure
- workflow termination
- workflow recovery

This engine ensures lifecycle transitions remain deterministic, policy-compliant, and auditable.

Lifecycle decisions are produced by the engine, while persistence is handled by system stores.

------------------------------------------------------------
LIFECYCLE STATE MODEL
------------------------------------------------------------

WorkflowLifecycleStatus values:

Created
Running
Waiting
Retrying
Completed
Failed
Terminated

------------------------------------------------------------
TRANSITION RULES
------------------------------------------------------------

Allowed transitions:

Created → Running (via Start)
Running → Completed (via Complete)
Running → Failed (via Fail)
Running → Terminated (via Terminate)
Waiting → Running (via Start)
Failed → Retrying (via Recover)
Failed → Terminated (via Terminate)
Retrying → Running (via Start)

Invalid transitions are rejected with reason.
