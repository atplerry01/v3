SECTION 1 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

You are implementing the **Merkle Proof Engine** for the WhyceChain system.

WhyceChain provides immutable evidence recording for Whycespace governance and economic operations.

Merkle proofs allow verification that a ledger entry belongs to a block without requiring access to all entries.

This enables:

• lightweight verification
• cross-system trust
• efficient audit validation
• distributed proof verification

The engine generates and validates **Merkle inclusion proofs**.

SECTION 6 — MERKLE TREE RULES
------------------------------------------------------------

Merkle tree must be constructed using:

Hash Algorithm: SHA256
Leaf nodes: ledger entry hashes
Internal nodes: SHA256(left + right)

If the number of nodes at a level is odd: duplicate the last node.

The Merkle root must match the block's stored MerkleRoot.
