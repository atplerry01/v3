# STEP 3 — PURPOSE OF CAPITAL CONTRIBUTION ENGINE

A **capital contribution** represents the moment funds actually enter the economic system.

Examples:

- investor transfers funds to capital pool
- CWG members contribute pooled capital
- institutional partner deposits investment funds

Characteristics:

- real monetary value transferred
- linked to a commitment (optional)
- recorded for pool balance calculations
- used for allocation into SPVs or assets

The engine must emit deterministic events for all contributions.

# STEP 7 — ENGINE IMPLEMENTATION

CapitalContributionEngine.cs

Responsibilities:

- validate contribution commands
- enforce deterministic contribution rules
- produce contribution events
- return immutable results

The engine is stateless, deterministic, thread-safe.

The engine does NOT persist data, call other engines, or access infrastructure directly.

Execution contract: Command -> Engine Execution -> Result -> Domain Event
