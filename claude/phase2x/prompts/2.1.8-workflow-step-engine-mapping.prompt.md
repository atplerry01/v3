COMPONENT PURPOSE
------------------------------------------------------------

Maps workflow steps to execution engines. Validates that each step references
exactly one engine and produces a resolved mapping structure for runtime orchestration.
This engine does NOT execute steps or invoke engines — it only resolves mappings.

------------------------------------------------------------
ENGINE CONTRACT
------------------------------------------------------------

IEngine implementation with EngineManifest attribute:
- Name: WorkflowStepEngineMapping
- Tier: T1M
- Kind: Decision
- Action: resolveMapping

Input: WorkflowStepEngineMappingCommand (WorkflowId, WorkflowSteps)
Output: EngineResult with resolved mappings

------------------------------------------------------------
VALIDATION RULES
------------------------------------------------------------

- WorkflowId must not be empty
- At least one workflow step required
- Each step must have non-empty StepId, StepName, EngineName
- Duplicate step IDs are rejected

------------------------------------------------------------
ARCHITECTURE CONSTRAINTS
------------------------------------------------------------

Engine is sealed, stateless, deterministic, thread-safe.
No persistence, no infrastructure access, no engine-to-engine calls.
