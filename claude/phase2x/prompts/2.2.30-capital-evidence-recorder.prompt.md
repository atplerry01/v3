# Component 2.2.30 — Capital Evidence Recorder

## System Category
Capital System

## Architecture Layer
System Layer

## Component Category
Evidence Integration Component

## Purpose
The Capital Evidence Recorder generates cryptographic evidence records for capital transactions
and anchors them into the Whycespace evidence infrastructure via WhyceChain.

## Evidence Operations
- CommitmentEvidence
- ContributionEvidence
- ReservationEvidence
- AllocationEvidence
- UtilizationEvidence
- DistributionEvidence

## Implementation Path
src/system/midstream/capital/evidence/

## Files
- CapitalEvidenceOperationType.cs — Evidence operation type enum
- CapitalEvidenceRecord.cs — Immutable evidence record model
- CapitalEvidenceHashUtility.cs — SHA256 deterministic hash generation
- ICapitalEvidenceRecorder.cs — Recorder interface with async methods
- CapitalEvidenceRecorder.cs — Thread-safe recorder implementation

## Tests
tests/capital-system/CapitalEvidenceRecorderTests.cs

## Debug Endpoint
GET /dev/capital/evidence/{capitalId}?referenceId={optional}

## Evidence Flow
Capital Transaction → Ledger Entry Created → Capital Evidence Record Generated → Evidence Hash Created → WhyceChain Ledger Entry Anchored

## Data Integrity Rules
- Deterministic SHA256 hash generation
- Immutable evidence records (sealed record)
- Traceable ledger reference via LedgerEntryId
- Unique EvidenceId enforced
- Corrections produce new evidence entries (no modification)

## Hash Input Fields
CapitalId | PoolId | ReferenceId | Amount | Currency | Timestamp
