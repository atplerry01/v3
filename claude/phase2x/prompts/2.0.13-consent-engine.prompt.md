# CONSENT ENGINE

This prompt must be executed using the Whycespace MASTER PROMPT GENERATION TEMPLATE.

This component implements the Consent Engine responsible for managing identity consent within the Whycespace ecosystem.

Consent represents explicit permission granted by an identity for specific data usage, system operations, or policy domains.

Consent management supports regulatory compliance (GDPR-style consent control) and internal policy governance.

Consent operations include:

consent granting
consent withdrawal
consent validation
consent scope evaluation

All architecture rules defined in:

claude/bootstrap-prompts/master-bootstrap.prompt.md

must be respected.

SECTION 2 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

Consent Engine performs operational consent lifecycle management.

Responsibilities:

• grant consent
• withdraw consent
• validate consent scope
• verify consent status
• emit consent events

Consent Engine must NOT:

• modify identity core data
• access infrastructure directly
• call external services
• perform persistence operations directly

The engine produces consent lifecycle decisions and events only.
