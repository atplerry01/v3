# WHYCESPACE WBSM v3
# CAPITAL SYSTEM
# COMPONENT 2.2.29 — CAPITAL LEDGER STORE

You are implementing the **Capital Ledger Store**.

The Capital Ledger Store persists **immutable financial ledger entries for all capital transactions** within the Whycespace Capital System.

The ledger records every capital movement, including:

- commitments
- contributions
- reservations
- allocations
- utilizations
- distributions

The ledger provides the **source of truth for financial traceability** within the capital infrastructure.

All ledger entries must be **immutable, append-only, and auditable**.

## Ledger Entry Model

CapitalLedgerEntry record with fields:
EntryId, EntryType, CapitalId, PoolId, InvestorIdentityId, ReferenceId,
Amount, Currency, PreviousBalance, NewBalance, Timestamp, TraceId, CorrelationId

LedgerEntryType enum:
CommitmentRecorded, ContributionRecorded, ReservationRecorded,
AllocationRecorded, UtilizationRecorded, DistributionRecorded, DistributionReversal

## Store Interface

ICapitalLedgerStore with methods:
- AppendEntry(CapitalLedgerEntry entry)
- GetEntriesByCapitalId(Guid capitalId)
- GetEntriesByPoolId(Guid poolId)
- GetEntriesByInvestor(Guid investorIdentityId)
- GetEntriesByReferenceId(Guid referenceId)
- GetLedgerRange(DateTimeOffset startDate, DateTimeOffset endDate)

## Data Integrity Rules

- Append-only writes
- Immutable entries
- Deterministic timestamps
- Traceability via TraceId and CorrelationId
- No updates or deletes — corrections via reversal entries only

## Thread Safety

- Lock-based synchronization for all read/write operations
- Index-based retrieval for efficient querying
