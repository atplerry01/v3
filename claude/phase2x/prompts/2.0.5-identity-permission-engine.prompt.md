# IDENTITY PERMISSION ENGINE

This prompt must be executed using the Whycespace MASTER PROMPT GENERATION TEMPLATE.

This component implements the Identity Permission Engine responsible for assigning and revoking permissions for identities.

Permissions define the allowed actions an identity can perform across the Whycespace ecosystem.

Permissions are typically derived from roles but may also be granted explicitly.

All architecture rules defined in:

claude/bootstrap-prompts/master-bootstrap.prompt.md

must be respected.

SECTION 2 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

Identity Permission Engine performs controlled mutation of identity permissions.

Responsibilities:

• grant permissions to identities
• revoke permissions from identities
• validate permission assignments
• emit permission change events

The engine must NOT:

• persist data
• access infrastructure directly
• call other engines

Persistence occurs through system stores.
