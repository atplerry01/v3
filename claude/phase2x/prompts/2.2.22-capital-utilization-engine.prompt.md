# COMPONENT 2.2.22 — CAPITAL UTILIZATION ENGINE

## Component Details

- **Component Number:** 2.2.22
- **Component Name:** Capital Utilization Engine
- **System Category:** Capital System
- **Architecture Layer:** Engine Layer
- **Engine Tier:** T2E — Execution Engine
- **Component Category:** EXECUTION ENGINE

## Purpose

Capital utilization represents the execution of allocated capital — the moment allocated capital is consumed by real economic activity such as SPV property acquisition, infrastructure deployment, asset purchase, or operational program spending.

## Repository Location

- **Engine:** `src/engines/T2E/core/capital/CapitalUtilizationEngine.cs`
- **Tests:** `tests/capital-system/CapitalUtilizationEngineTests.cs`
- **Domain Models:** `src/domain/economic/capital/`
- **System Components:** `src/system/midstream/capital/registry/`

## Operations

### UtilizeCapital
Records the deployment of allocated capital against a target (SPV, Asset, Project, OperationalProgram).

### AdjustUtilization
Adjusts a previously recorded utilization amount with a reason.

### ReverseUtilization
Reverses a utilization record with a reason.

## Domain Events

- `CapitalUtilized` — capital deployed against target
- `AllocationConsumed` — linked allocation marked as consumed
- `CapitalUtilizationAdjusted` — utilization amount adjusted
- `CapitalUtilizationReversed` — utilization reversed

## Governance Integration

Runtime pipeline enforces:
- Identity Verification via WhyceID
- Policy Evaluation via WhycePolicy (allocation authorization, spending limits, SPV spending authority, project budget constraints)

## Architecture Compliance

- Stateless, deterministic, thread-safe
- No persistence inside engine
- No engine-to-engine calls
- Events published through runtime EventPublisher
- Topic: `whyce.capital.events`
