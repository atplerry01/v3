# AUTHORIZATION ENGINE

This prompt must be executed using the Whycespace MASTER PROMPT GENERATION TEMPLATE.

This component implements the Authorization Engine responsible for evaluating whether an authenticated identity is permitted to perform a specific action on a resource within a defined scope.

Authorization decisions enforce access control across the Whycespace ecosystem.

Authorization evaluates:

identity roles
permissions
access scopes
policy constraints
trust score requirements

All architecture rules defined in:

claude/bootstrap-prompts/master-bootstrap.prompt.md

must be respected.

SECTION 2 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

Authorization Engine performs operational access control evaluation.

Responsibilities:

• evaluate access permission
• validate role assignments
• validate permission scope
• enforce policy restrictions
• produce authorization decision

Authorization Engine must NOT:

• mutate identity state
• persist system data
• access infrastructure directly
• call external services

It produces authorization decisions only.

SECTION 6 — AUTHORIZATION RULES
------------------------------------------------------------

Identity is authenticated

Required permission exists

Scope allows requested action

Trust score meets minimum requirement

Device trust score acceptable
