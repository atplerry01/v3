## Purpose

The Capital Registry records all registered capital entities including pools, commitments,
contributions, reservations, allocations, and distributions. It is the authoritative directory
of capital objects. It does NOT perform business logic.

## Architecture Compliance

- System layer placement (not engine, not domain)
- No domain models defined — references domain types only
- No business logic — pure registration and retrieval
- Thread-safe via lock-based synchronization
- Infrastructure agnostic via ICapitalRegistryStore abstraction
- Does not publish events — projection workers update the registry
