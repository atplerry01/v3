SECTION 1 — ARCHITECTURE VERIFICATION
------------------------------------------------------------

You are implementing the **Chain Evidence Gateway** for the WhyceChain system.

WhyceChain provides immutable evidence recording for governance and economic operations in Whycespace.

The Chain Evidence Gateway acts as the **platform access gateway** used by other systems to interact with the WhyceChain evidence infrastructure.

Systems that may use this gateway include:

- WhycePolicy
- WhyceID
- Workflow systems (WSS)
- Cluster governance systems
- Runtime orchestration components

The gateway receives evidence submissions and verification requests and routes them into the WhyceChain processing pipeline.

The gateway does NOT perform hashing, block building, or anchoring itself.

Those responsibilities belong to engines.

SECTION 3 — COMPONENT RESPONSIBILITY
------------------------------------------------------------

The Chain Evidence Gateway provides the **entry point for evidence submission and verification**.

Responsibilities:

- receive evidence submission requests
- validate request structure
- route requests to the appropriate engine pipeline
- return evidence references
- support evidence verification queries

The gateway does NOT:

- execute hashing algorithms
- build blocks
- perform anchoring
- access infrastructure directly

SECTION 6 — GATEWAY OPERATIONS
------------------------------------------------------------

SubmitEvidence(EvidenceSubmissionRequest request):

1. validate request format
2. forward request to evidence processing pipeline
3. receive evidence hash
4. return EvidenceSubmissionResponse

VerifyEvidence(EvidenceVerificationRequest request):

1. validate request structure
2. request verification from chain integrity systems
3. return verification result
