------------------------------------------------------------
SYSTEM CONTEXT
------------------------------------------------------------

The **Identity Recovery Engine** manages secure recovery workflows for identities
that have lost authentication access.

Recovery scenarios include:

• credential loss
• device loss
• compromised identity
• account lockout
• guardian-assisted recovery

The engine evaluates recovery requests and produces deterministic recovery
decisions.

The engine must not mutate domain aggregates.

Instead it produces **IdentityRecoveryEvents** that are later handled by workflows.

ENGINE RESPONSIBILITIES
------------------------------------------------------------

The engine must evaluate identity recovery requests and determine
whether recovery is allowed.

The engine produces deterministic results for:

• recovery eligibility
• verification requirement
• recovery risk level
• recovery approval

The engine must compute a deterministic **RecoveryId** using SHA256.

------------------------------------------------------------
ENGINE EXECUTION
------------------------------------------------------------

The engine performs the following evaluation steps:

1. Validate identity existence
2. Validate identity status
3. Evaluate verification evidence
4. Evaluate recovery method risk level
5. Evaluate recovery eligibility
6. Determine recovery approval

The engine must remain **pure and deterministic**.
