## Summary

Refactored the Whycespace simulation system from a standalone console app (`simulation/Whycespace.Simulation/`) into an enterprise-grade platform system at `src/platform/simulation/`.

## What Changed

### Structure Created

```

├── Program.cs                                    (CLI entry point)
├── Whycespace.Platform.Simulation.csproj          (project file)
├── config/
│   └── SimulationConfiguration.cs                 (global config)
├── dispatch/
│   └── SimulationEngineDispatcher.cs              (IEngineRuntimeDispatcher impl)
├── event-capture/
│   ├── SimulationEventCollector.cs                (NEW — captures events without publishing)
│   └── SimulationEventGenerator.cs                (moved from simulation/)
├── fault-injection/
│   ├── ChaosEngine.cs                             (NEW — enterprise chaos engineering)
│   └── FaultInjector.cs                           (moved from simulation/)
├── metrics/
│   └── SimulationMetrics.cs                       (moved from simulation/)
├── orchestration/

├── reporting/

├── scenarios/
│   └── SimulationScenario.cs                      (moved from simulation/)
└── trace/
```

### New Components

1. **SimulationOrchestrator** — Cross-layer simulation orchestrator integrating policy, event capture, and trace collection
2. **SimulationEventCollector** — Thread-safe event capture without publishing to event fabric
3. **SimulationPolicyAdapter** — WhycePolicy integration adapter with local allow/block lists
4. **SimulationTraceCollector** — Execution trace collection for audit and debugging
5. **ChaosEngine** — Enterprise chaos engineering with targeted rules (latency, failure, timeout)
6. **SimulationConfiguration** — Global simulation config record
