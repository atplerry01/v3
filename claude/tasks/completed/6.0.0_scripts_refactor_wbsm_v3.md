# 6.0.0 — Scripts Refactor (WBSM v3 STRICT)

**Status:** COMPLETED
**Date:** 2026-03-18
**Mode:** WBSM v3 STRICT

---

## Objective

Refactor `scripts/` from flat 3-folder structure into enterprise-grade, domain-organized operational layer.

## Before

```
scripts/
├── build/       (build.ps1, build.sh)
├── dev/         (start-dev.ps1, start-dev.sh)
└── tools/       (generate-engine.ps1, generate-workflow.ps1)
```

3 directories, 6 files.

## After

```
scripts/
├── build/                    build.ps1, build.sh
├── dev/                      start-dev.ps1, start-dev.sh
├── runtime/                  inspect-runtime.ps1
├── infrastructure/           (reserved)
├── persistence/              replay-events.ps1
├── simulation/               run-simulation.ps1
├── observability/            (reserved)
├── governance/               evaluate-policy.ps1
├── generators/               generate-engine.ps1, generate-workflow.ps1
├── ci/                       (reserved)
├── validation/               validate-system.ps1
└── shared/                   ScriptHelpers.psm1
```

12 directories, 12 files (6 existing refactored + 5 new scripts + 1 shared module).

## Changes Applied

### Structural
- Moved `tools/*` → `generators/*`
- Created 9 new domain directories
- All directories follow system domain organization

### New Scripts
| Script | Purpose |
|--------|---------|
| `shared/ScriptHelpers.psm1` | Shared module: banner, step logging, path resolution, exit code assertion |
| `runtime/inspect-runtime.ps1` | Scans engine registry, workflow definitions, partition configuration |
| `persistence/replay-events.ps1` | Event replay for projection/workflow rebuild with dry-run support |
| `simulation/run-simulation.ps1` | Parameterized simulation runner wrapping Whycespace.Simulation |
| `governance/evaluate-policy.ps1` | Policy validation, integration point scanning, governance coverage audit |
| `validation/validate-system.ps1` | Layer boundary, project reference, contract, naming convention validation |

### Standardization
- All `.ps1` scripts import `ScriptHelpers.psm1`
- Consistent `Write-Banner`, `Write-Step`, `Write-Success`, `Assert-ExitCode` pattern
- All scripts fully parameterized with `[ValidateSet]` where applicable
- Generator scripts updated: tier params simplified to `T0U`-`T4A`, corrected namespaces

### Reserved Directories
- `infrastructure/` — for infra provisioning scripts
- `observability/` — for metrics/tracing/alerting scripts
- `ci/` — for CI/CD pipeline scripts

## Validation
- All `Import-Module` paths verified (9/9 consistent)
- Directory structure verified (12 directories)
- No broken references
- No duplicate functionality
